Vanishing Array



You are given an array of n non-negative integers.
The integers are vanishing one by one for some unknown reason.
The order in which they vanish is given to you as a permutation of integers from 1 to n.
After each element vanishes you have to find out the maximum sum of unvanished elements, such that no vanished element is between.


You must fill in the solve function which takes in n, a list of n numbers, and a list of a permutation 1 to n.
The function must output a list of answers where the ith element corresponds to the answer after the first i steps.
The following conditions are guaranteed:

1 ≤ n ≤ 100,000

0 ≤ numbers[i] ≤ 1,000,000,000


Examples

Example 1

n = 4
numbers = [1,3,2,5]
perm = [3,4,1,2]
output = [5,4,3,0]

Example 2

n = 5
numbers = [1,2,3,4,5]
perm = [4,2,3,5,1]
output = [6,5,5,1,0]





Try to think backwards.
Instead of vanishing, start at the end. Then the integers are appearing one by one.
When an integer appears, you must check both sides to see if they are already here. And for each side present, the sum of this block is increased.
Blocks are disjoint sets of indices. Some are present. Some are not.
Consider a DSU with the numbers 1 - n. Whenever an integer appears, you must set it's DSU to present and for each of its sides, if it is present, you must unite these two and update this DSU's size and sum of integers in the original array.
Introduce a local variable as the answer from the end. Every time an integer appears, this answer is changed if the sum of the resulting DSU is larger than it.
When you are adding an integer, the new answer is the maximum of the previous answer and the sum of the currently added DSU.
