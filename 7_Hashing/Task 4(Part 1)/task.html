در این سوال می‌خواهیم با استفاده از داده‌ ساختار جدول درهم سازی یک LRU Cache پیاده سازی کنیم.
برای اینکار باید توابع کلاس LRU Cache را کامل کنید. هنگام ساخت LRU Cache یک عدد به عنوان اندازه آن (size) به سازنده آن پاس داده می‌شود. این cache به این شکل کار می‌کند که ابتدا به کمک add مقادیر را به صورت key-value در آن درج می‌کنیم و به کمک تابع get مقادیر متناظر با کلید‌هایی که قبلا درج کرده‌ایم را بازیابی می‌کنیم. هنگامی که cache پر شود آخرین داده استفاده شده یعنی داده‌ای که قبل از بقیه get یا add شده باشد پاک می‌شود. برای درک بهتر به مثال مراجعه کنید.

شما باید سه تابع add, get و init کلاس LRUCache را به گونه‌ای پیاده سازی کنید که هر عملیات add و get به طور متوسط در زمان (۱)O قابل انجام باشد. تابع add کلید key و مقدار value را دریافت کرده و آن را در cache درج می‌کند. تابع get در صورتیکه کلید داده شده در cache موجود باشد value متناظر با آن را برمی‌گرداند و در غیر این صورت مقدار None را برمی‌گرداند. تابع init نیز یک LRUCache با سایز مشخص شده می‌سازد.

نکته:
برای این سوال نیاز به پیاده سازی جدول درهم‌سازی دارید، خودتان این داده ساختار را در فایل hash_table.py پیاده سازی کنید. برای حل مشکل برخورد از روش آدرس‌دهی باز استفاده کنید.

مثال:
cache = LRUCache(3)
cache.add(23, 765)
cache.add(100, 100000)
cache.add(2, -309)
cache.get(23)
cache.get(2)
cache.put(0, 2)
print(cache.get(23))
print(cache.get(100))
print(cache.get(2))
print(cache.get(0))

Output:
765
None
0398
2
قاعدتا برای پیاده سازی نیاز به صف هم دارید!
دقت کنید ظرفیت برای اینکه عملیات get از مرتبه (۱)O باقی بماند باید ظرفیت جدول در هم سازی به مراتب بیشتر از ظرفیت cache باشد.
